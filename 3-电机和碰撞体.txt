default_dof_drive_mode = 3

  IsaacGym 关节驱动模式，对应 GymDofDriveModeFlags 枚举：

  ┌─────┬─────────────────┬──────────────────────────────────────────────┐
  │ 值  │      模式       │                     含义                     │
  ├─────┼─────────────────┼──────────────────────────────────────────────┤
  │ 0   │ DOF_MODE_NONE   │ 无驱动，关节自由运动                         │
  ├─────┼─────────────────┼──────────────────────────────────────────────┤
  │ 1   │ DOF_MODE_POS    │ 位置目标模式，PhysX 内置 PD 直接跟踪目标角度 │
  ├─────┼─────────────────┼──────────────────────────────────────────────┤
  │ 2   │ DOF_MODE_VEL    │ 速度目标模式，跟踪目标速度                   │
  ├─────┼─────────────────┼──────────────────────────────────────────────┤
  │ 3   │ DOF_MODE_EFFORT │ 力矩驱动模式，外部传入力矩值                 │
  └─────┴─────────────────┴──────────────────────────────────────────────┘

  设为 3（力矩驱动） 的原因：策略输出目标角度后，由 _compute_torques() 在 Python 侧自行计算 PD 力矩，再通过 set_dof_actuation_force_tensor() 直接施加到关节。这样做的好处是：
  - 可以叠加域随机化（p_gains_multiplier、d_gains_multiplier、motor_zero_offsets、motor_strengths）
  - 完全控制 PD 计算逻辑，方便加入电机零位偏移等真实机器人特性

  如果用模式 1，PhysX 内置 PD 不支持这些随机化，sim-to-real 迁移效果差。


thickness = 0.01

  碰撞体的额外"厚度"，单位米（即 1cm）。

  这是 IsaacGym 中 AssetOptions.thickness 参数，作用是给每个碰撞 shape 在接触检测时额外膨胀一层厚度，相当于给碰撞体包了一层 1cm 的壳。

  实际影响：
  - 足端接触检测更灵敏：足端碰撞球膨胀 1cm 后，在真正物理接触发生前就能检测到接触事件，减少漏检
  - 防止穿透：细长的腿部碰撞体加了厚度后，在高速运动时不容易穿过地形网格（trimesh）
  - 对奖励的影响：contact_forces[:, feet_indices, 2] > 1. 这类接触判断依赖 PhysX 能正确上报接触力，thickness 保证了接触力上报的稳定性

  当前两个任务的值：

  ┌────────────────────────┬────────────────────┬────────────────────┐
  │          参数          │    go2_moe_cts     │   boying_moe_cts   │
  ├────────────────────────┼────────────────────┼────────────────────┤
  │ default_dof_drive_mode │ 3（继承基类）      │ 3（继承基类）      │
  ├────────────────────────┼────────────────────┼────────────────────┤
  │ thickness              │ 0.01 m（继承基类） │ 0.01 m（继承基类） │
  └────────────────────────┴────────────────────┴────────────────────┘

  两个任务都没有覆写，直接使用基类默认值。一般不需要修改这两个参数，除非遇到足端接触力上报不稳定或关节驱动方式需要切换的特殊情况。
